# 🚀 DigiPlay Cloud Deployment Guide (AWS)

This guide provides a comprehensive, professional walkthrough for deploying your DigiPlay backend (Express Server + MQTT Broker) to the AWS Cloud. 

---

## 💰 1. Cost Analysis (AWS Free Tier)

If you are using an AWS Free Tier eligible account, you can host this entire system for **$0.00 / month** for the first 12 months.

| Component | AWS Service | Free Tier Allowance | Out-of-Tier Estimate |
| :--- | :--- | :--- | :--- |
| **Compute** | EC2 `t2.micro` | 750 hours / month | ~$8.50 / month |
| **Storage** | EBS General Purpose SSD | 30 GB | ~$3.00 / month |
| **IP Address** | Elastic IP | Free (while instance is running) | ~$0.005 / hr (if idle) |
| **Database** | SQLite (On-disk) | Included in EBS storage | $0.00 |
| **Data Transfer**| In/Out bandwidth | 100 GB / month | ~$0.09 / GB |

> [!IMPORTANT]
> To avoid accidental charges, ensure you select **Ubuntu 22.04 LTS** and the **t2.micro** instance type, both marked as "Free Tier Eligible".

## 🛠️ 2. Phase 1: Launching the Virtual Server (EC2)

1.  **Login**: Go to the [AWS EC2 Console](https://console.aws.amazon.com/ec2/).
2.  **Launch**: Click the orange **"Launch instance"** button.
3.  **Names and Tags**: Set Name to `DigiPlay-Production-Server`.
4.  **OS Image**: Select **Ubuntu** -> **Ubuntu Server 22.04 LTS (HVM)**.
5.  **Instance Type**: Ensure `t2.micro` is selected.
6.  **Key Pair**: 
    *   Click "Create new key pair".
    *   Name it `digiplay_key`.
    *   Format: `.pem` (for OpenSSH) or `.ppk` (for PuTTY on Windows).
    *   **CRITICAL**: Download and save this file. You cannot download it again later.
7.  **Network Settings**:
    *   Check "Allow SSH traffic from Anywhere (0.0.0.0/0)".
    *   Check "Allow HTTP traffic from the internet".
8.  **Launch**: Click **Launch instance** at the bottom right.

---

## 🔒 3. Phase 2: Configuring the Firewall (Security Groups)

By default, AWS blocks the ports needed for MQTT and your custom API.

1.  On the EC2 Dashboard, click **"Instances (running)"**.
2.  Select your instance and look at the **"Security"** tab below.
3.  Click the link under **"Security groups"** (e.g., `sg-0a1b2c...`).
4.  Click **"Edit inbound rules"**.
5.  Add the following rules:

| Protocol | Port Range | Source | Description |
| :--- | :--- | :--- | :--- |
| **TCP** | `8000` | `0.0.0.0/0` | Web Dashboard & Mobile API |
| **TCP** | `1883` | `0.0.0.0/0` | MQTT Broker (Hardware Connection) |
| **TCP** | `80` | `0.0.0.0/0` | Standard HTTP (Optional) |

6.  Click **"Save rules"**.

---

## 🌐 4. Phase 3: Setting a Static IP (Elastic IP)

AWS changes your server's IP address every time you restart it. An **Elastic IP** gives you a permanent address so your ESP32 never loses connection.

1.  In the left sidebar of the EC2 Console, scroll down to **Network & Security** -> **Elastic IPs**.
2.  Click **"Allocate Elastic IP address"** -> **Allocate**.
3.  Select the new IP, click **Actions** -> **Associate Elastic IP address**.
4.  Select your `DigiPlay-Production-Server` instance and click **Associate**.
5.  **Note this IP address**; this is your permanent Server Address.

## 💻 5. Phase 4: Server Environment Setup

### 5.1 Connect via SSH
Open your local terminal and run:
```bash
chmod 400 digiplay_key.pem  # Secure the key
ssh -i "digiplay_key.pem" ubuntu@YOUR_ELASTIC_IP
```

### 5.2 Install System Dependencies
Once logged in, run these commands:
```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs git

# Verify installation
node -v && npm -v
```

### 5.3 Deploy Codebase
```bash
git clone <YOUR_GIT_REPOSITORY_URL>
cd Version_3/server
npm install
```

### 5.4 Configure Environment
Create the production config:
```bash
nano .env
```
Paste and edit these values:
```env
PORT=8000
SECRET_KEY=  # Generate one using: openssl rand -hex 32
APP_API_KEY=your_secure_mobile_app_key
ENVIRONMENT=production
```
Save and exit (`Ctrl+O`, `Enter`, `Ctrl+X`).

## 🚀 6. Phase 5: Production Process Management (PM2)

In production, you don't use `node index.js`. You use **PM2** to ensure the server auto-restarts if it crashes or if the AWS hardware reboots.

```bash
# Install PM2 globally
sudo npm install -g pm2

# Initialize Database
npm run seed

# Start the server
pm2 start index.js --name "digiplay-backend"

# Configure Auto-Start on Reboot
pm2 startup
# (Copy and paste the command generated by the line above)
pm2 save
```

---

## 🔌 7. Phase 6: Hardware Configuration

Update your ESP32 code (`digiplay_firmware.ino`) to point to the cloud:

1.  **Update Broker**: Change `MQTT_BROKER` to your **Elastic IP**.
2.  **Check Ports**: Ensure `MQTT_PORT` is `1883`.
3.  **Upload**: Flash the firmware to your device.

---

## 🔍 8. Troubleshooting

*   **ESP32 not connecting?**: Check the AWS Security Group rules. Port `1883` must be open.
*   **Dashboard not loading?**: Visit `http://YOUR_IP:8000/dashboard`. Ensure Port `8000` is open.
*   **Logs**: To see what the server is doing in real-time, run `pm2 logs digiplay-backend`.

---

## 🛑 9. How to stop everything
If you want to stop the server to avoid any charges:
1.  Go to EC2 Console -> Instances.
2.  Select the instance -> **Instance State** -> **Terminate**. (Warning: This deletes the server).
3.  **IMPORTANT**: Go to **Elastic IPs**, select your IP, click **Actions** -> **Release Elastic IP address**. AWS charges for Elastic IPs that are NOT attached to a running server.
